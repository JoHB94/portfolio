📝 요구사항 ​

📋 리스트 기능 ​

  1. 클릭 시 이동 기능

    • 로그인 화면으로 이동
    • 회원가입 화면으로 이동
    • 글쓰기 화면으로 이동

  2. 게시글 정보 출력

    • 게시글 갯수 출력
    • 등록된 게시글의 TYPE 출력 (일반, 자유, Q&A, 익명)
    • 게시글 No 출력
    • 게시글 TITLE 출력

  3. 조회 및 페이징 기능

    • Check box 선택 후 조회 버튼 클릭 시 조건에 맞게 출력
    • 한 페이지에 10개의 게시글씩 출력되도록 paging 처리

🔑 회원가입 ​

  1. 회원가입 후 이동

    • 회원가입 완료 후 메인 리스트 화면으로 이동

  2. ID 중복 체크

    • DB에 있는 기존 아이디들과 중복 체크
    • ID 중복 미확인 시 회원가입 불가 (alert 또는 중복 메시지 출력)

  3. 회원가입 필수 입력 값

    • ID, PW, PW CHECK, name, phone

  4. 입력 규칙

    • phone → 첫 번째 select box의 값은 DB에서 출력
    • PW → 6~12자리 & PW CHECK 값과 동일해야 함
    • phone text 입력 → 각각 4자리
    • postNo (우편번호) → xxx-xxx 형식으로 작성

🔓 로그인 ​

  1. 로그인 입력 필드

    • ID 입력
    • PW 입력

  2. 로그인 버튼 클릭 시 동작

    • ID 및 PW 유효성 체크
    • ID 또는 PW가 틀릴 경우 alert 창에 메시지 출력

  3. 로그인 상태 UI 변화

    • 로그인 시 사용자의 name 출력
    • Join & login 링크 숨김 처리
    • 로그인 시에만 logout 버튼 표시

✍ 글 작성 ​

  1. 글 저장 및 이동

    • 작성 완료 후 글이 저장되며 메인 화면 리스트에 출력
    • 저장 후 메인 화면으로 자동 이동

  2. 게시글 Type 지정

    • DB의 code_name을 지정하고, 저장 시 code_id 값을 저장

  3. 로그인한 사용자의 정보 출력

    • 작성자 name 출력

📑 글 상세 ​

  1. 작성자와 로그인 ID가 일치할 경우

    • Delete / Update 버튼 활성화

  2. 버튼 동작

    • Update 버튼 클릭 → 수정 페이지로 이동
    • Delete 버튼 클릭 → 게시글 삭제 후 리스트로 이동
